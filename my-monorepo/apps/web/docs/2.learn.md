## 🧠 ahooks 核心原理深度解析

ahooks 是阿里巴巴开源的高质量 React Hooks 库（[GitHub 28k+ stars](https://github.com/alibaba/hooks)），核心思想是 **"封装副作用 + 状态管理 + 性能优化"**。下面按类别拆解核心 hooks 的实现原理。

---

## 📦 一、状态管理类 Hooks

### 1️⃣ `useBoolean` / `useToggle`

#### 原理
封装 `useState<boolean>`，提供语义化 API（`toggle/setTrue/setFalse`）

#### 简化源码
```typescript
function useBoolean(defaultValue = false) {
  const [state, setState] = useState(defaultValue);
  
  const actions = useMemo(() => {
    const toggle = (value?: boolean) => 
      setState(prev => (typeof value === 'boolean' ? value : !prev));
    
    const setTrue = () => setState(true);
    const setFalse = () => setState(false);
    
    return { toggle, setTrue, setFalse };
  }, []);
  
  return [state, actions] as const;
}
```

#### 使用场景
```tsx
const [visible, { toggle, setTrue, setFalse }] = useBoolean(false);

return (
  <>
    <button onClick={toggle}>切换</button>
    <button onClick={setTrue}>显示</button>
    <Modal visible={visible} onClose={setFalse} />
  </>
);
```

---

### 2️⃣ `useSet` / `useMap`

#### 原理
用 `useState` + `useMemo` 封装 Set/Map 操作，**返回稳定引用的 action 函数**（避免 useEffect 无限循环）

#### 简化源码（useSet）
```typescript
function useSet<T>(initialValue?: Iterable<T>) {
  const [set, setSet] = useState<Set<T>>(() => 
    initialValue ? new Set(initialValue) : new Set<T>()
  );
  
  const actions = useMemo(() => {
    const add = (key: T) => setSet(prev => {
      const next = new Set(prev);
      next.add(key);
      return next;
    });
    
    const remove = (key: T) => setSet(prev => {
      const next = new Set(prev);
      next.delete(key);
      return next;
    });
    
    const reset = () => setSet(new Set());
    
    return { add, remove, reset };
  }, []);
  
  return [set, actions] as const;
}
```

#### ⚠️ 关键设计
- **不可变更新**：每次操作都创建新 Set/Map（触发 React 重新渲染）
- **useMemo 包装 actions**：确保函数引用稳定，避免 useEffect 依赖问题

## ⚙️ 二、副作用增强类 Hooks

### 3️⃣ `useUpdateEffect`（跳过首次渲染）

#### 原理
用 `useRef` 记录是否首次渲染，**跳过 mount 阶段**，只在 update 时执行

#### 简化源码
```typescript
function useUpdateEffect(effect: EffectCallback, deps?: DependencyList) {
  const isMounted = useRef(false);
  
  useEffect(() => {
    if (!isMounted.current) {
      isMounted.current = true; // 首次渲染标记
      return;
    }
    return effect(); // 后续更新才执行
  }, deps);
}
```

#### 使用场景
```tsx
// 表单值变化时保存，但首次加载不保存
useUpdateEffect(() => {
  saveToLocalStorage(formValues);
}, [formValues]);
```

---

### 4️⃣ `useDeepCompareEffect`（深度比较依赖）

#### 原理
用 `useRef` + `useEffect` + `isEqual`（lodash）实现深度比较，**避免对象/数组引用变化导致的无限循环**

#### 简化源码
```typescript
function useDeepCompareEffect(effect: EffectCallback, deps: DependencyList) {
  const ref = useRef<DependencyList>();
  
  // 深度比较依赖是否变化
  const isChanged = useMemo(() => {
    if (ref.current) {
      return !isEqual(ref.current, deps);
    }
    return true; // 首次渲染视为变化
  }, [deps]);
  
  // 保存最新依赖
  useEffect(() => {
    ref.current = deps;
  });
  
  // 只有深度比较变化时才执行 effect
  useEffect(() => {
    if (isChanged) {
      return effect();
    }
  }, [isChanged]);
}
```

#### ⚠️ 性能警告
- 深度比较有性能开销，**仅在必要时使用**
- 优先考虑 `useMemo` 优化依赖本身

## 🌐 三、DOM 操作类 Hooks

### 5️⃣ `useEventListener`（安全事件监听）

#### 原理
封装 `addEventListener`/`removeEventListener`，**自动处理组件生命周期 + ref 变化**

#### 简化源码
```typescript
function useEventListener<K extends keyof WindowEventMap>(
  eventName: K,
  handler: (event: WindowEventMap[K]) => void,
  options?: boolean | AddEventListenerOptions
): void;

function useEventListener<K extends keyof HTMLElementEventMap>(
  eventName: K,
  handler: (event: HTMLElementEventMap[K]) => void,
  target?: RefObject<HTMLElement> | HTMLElement,
  options?: boolean | AddEventListenerOptions
): void;

function useEventListener(
  eventName: string,
  handler: Function,
  target: any = window,
  options?: boolean | AddEventListenerOptions
) {
  const handlerRef = useRef<Function>();
  
  // 保存最新 handler（避免闭包问题）
  useEffect(() => {
    handlerRef.current = handler;
  }, [handler]);
  
  useEffect(() => {
    // 获取目标元素
    const targetElement = target?.current ?? target;
    if (!targetElement?.addEventListener) return;
    
    // 代理函数（始终调用最新 handler）
    const eventListener = (event: Event) => 
      handlerRef.current?.(event);
    
    // 监听事件
    targetElement.addEventListener(eventName, eventListener, options);
    
    // 清理
    return () => {
      targetElement.removeEventListener(eventName, eventListener, options);
    };
  }, [eventName, target, options]);
}
```

#### 核心优势
- ✅ 自动清理（避免内存泄漏）
- ✅ 支持动态 `ref`（target 变化时自动重新绑定）
- ✅ 闭包安全（通过 `ref` 保存最新 handler）

#### 使用场景
```tsx
const MyComponent = () => {
  const ref = useRef(null);
  
  // 窗口 resize
  useEventListener('resize', handleResize);
  
  // 元素 click
  useEventListener('click', handleClick, ref);
  
  return <div ref={ref}>Click me</div>;
};
```

### 6️⃣ `useClickAway`（点击外部区域）

#### 原理
监听 `mousedown` + `mouseup` 事件，**判断点击目标是否在元素外部**

#### 简化源码
```typescript
function useClickAway(
  ref: RefObject<HTMLElement>,
  onClickAway: (event: MouseEvent | TouchEvent) => void,
  eventName: 'click' | 'mousedown' | 'mouseup' = 'click'
) {
  const handler = useCallback((event: MouseEvent | TouchEvent) => {
    const element = ref.current;
    if (!element) return;
    
    // 检查点击目标是否在元素外部
    if (!element.contains(event.target as Node)) {
      onClickAway(event);
    }
  }, [ref, onClickAway]);
  
  useEventListener(eventName, handler);
}
```

#### 使用场景
```tsx
const Dropdown = () => {
  const ref = useRef(null);
  const [open, setOpen] = useState(false);
  
  useClickAway(ref, () => setOpen(false));
  
  return (
    <div ref={ref}>
      <button onClick={() => setOpen(!open)}>Toggle</button>
      {open && <Menu />}
    </div>
  );
};
```

## ⚡ 四、性能优化类 Hooks

### 7️⃣ `useMemoizedFn`（持久化函数）

#### 原理
用 `useRef` 保存函数引用，**返回始终不变的函数**（解决 useCallback 依赖问题）

#### 简化源码
```typescript
function useMemoizedFn<T extends (...args: any[]) => any>(fn: T) {
  const fnRef = useRef<T>(fn);
  
  // 始终更新 ref 中的最新函数
  fnRef.current = fn;
  
  // 返回一个稳定引用的函数
  const memoizedFn = useRef<T>();
  if (!memoizedFn.current) {
    memoizedFn.current = function (...args) {
      return fnRef.current.apply(this, args);
    } as T;
  }
  
  return memoizedFn.current;
}
```

#### 为什么需要？
```tsx
// ❌ 问题：callback 每次渲染都变化，导致子组件重复渲染
const callback = useCallback(() => {
  console.log(count); // 依赖 count
}, [count]);

// ✅ 解决：useMemoizedFn 返回稳定引用，内部自动捕获最新 count
const callback = useMemoizedFn(() => {
  console.log(count); // 始终访问最新 count
});
```

#### 核心优势
- ✅ 函数引用永不变化（`callback === callback` 始终为 true）
- ✅ 内部自动捕获最新闭包值
- ✅ 完美解决 useEffect/useCallback 依赖地狱

### 8️⃣ `useThrottle` / `useDebounce`

#### 原理
结合 `useState` + `useEffect` + `lodash.throttle/debounce`，**自动清理定时器**

#### 简化源码（useDebounce）
```typescript
function useDebounce<T>(value: T, delay: number): T {
  const [debouncedValue, setDebouncedValue] = useState(value);
  
  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);
    
    return () => clearTimeout(handler); // 自动清理
  }, [value, delay]);
  
  return debouncedValue;
}
```

#### 高级版：支持 options
```typescript
function useDebounce<T>(
  value: T, 
  options?: { 
    wait?: number; 
    leading?: boolean; 
    trailing?: boolean;
  }
) {
  const { wait = 1000, leading = false, trailing = true } = options || {};
  const [debounced, setDebounced] = useState(value);
  const debouncedRef = useRef(value);
  
  useEffect(() => {
    debouncedRef.current = value;
  }, [value]);
  
  useEffect(() => {
    const handler = setTimeout(() => {
      if (trailing) {
        setDebounced(debouncedRef.current);
      }
    }, wait);
    
    if (leading && debouncedRef.current !== debounced) {
      setDebounced(debouncedRef.current);
    }
    
    return () => clearTimeout(handler);
  }, [wait, leading, trailing]);
  
  return debounced;
}
```

## 🌐 五、网络请求类 Hooks

### 9️⃣ `useRequest`（SWR 模式）

#### 原理
实现 **SWR（Stale-While-Revalidate）** 模式：
1. 先返回缓存数据（stale）
2. 后台重新验证（revalidate）
3. 更新最新数据

#### 核心状态机
```typescript
type RequestState<T> = {
  loading: boolean;   // 是否加载中
  data?: T;           // 成功数据
  error?: Error;      // 错误
  params?: any[];     // 最后一次请求参数
  run: (...params: any[]) => Promise<T>; // 手动触发
  refresh: () => void; // 重新验证
  mutate: (data?: T | ((oldData?: T) => T | undefined)) => void; // 手动修改缓存
};
```

#### 简化实现
```typescript
function useRequest<T>(
  service: (...args: any[]) => Promise<T>,
  options: { 
    manual?: boolean; 
    defaultParams?: any[]; 
    onSuccess?: (data: T) => void;
  } = {}
) {
  const [state, setState] = useState<RequestState<T>>({
    loading: !options.manual,
    data: undefined,
    error: undefined,
  });
  
  const serviceRef = useRef(service);
  serviceRef.current = service;
  
  const run = useCallback(async (...params: any[]) => {
    setState(prev => ({ ...prev, loading: true, error: undefined }));
    
    try {
      const data = await serviceRef.current(...params);
      setState({ loading: false, data, error: undefined, params });
      options.onSuccess?.(data);
      return data;
    } catch (error) {
      setState({ loading: false, data: state.data, error: error as Error, params });
      throw error;
    }
  }, []);
  
  // 首次自动执行
  useEffect(() => {
    if (!options.manual) {
      run(...(options.defaultParams || []));
    }
  }, []);
  
  return {
    ...state,
    run,
    refresh: () => run(...(state.params || [])),
    mutate: (data: T | ((old?: T) => T)) => {
      setState(prev => ({
        ...prev,
        data: typeof data === 'function' ? data(prev.data) : data,
      }));
    },
  };
}
```

#### SWR 特性
- ✅ 自动缓存（相同参数返回缓存数据）
- ✅ 后台刷新（focus/tab 切换时自动 revalidate）
- ✅ 轮询（pollingInterval）
- ✅ 分页/加载更多（loadMore）

## 🧪 六、表单类 Hooks

### 🔟 `useForm`（简化版）

#### 原理
用 `useReducer` 管理表单状态，提供 `getFieldValue/setFieldsValue/validate` 等 API

#### 核心设计
```typescript
type FormAction =
  | { type: 'SET_FIELD'; field: string; value: any }
  | { type: 'SET_FIELDS'; fields: Record<string, any> }
  | { type: 'VALIDATE'; callback: (errors: Record<string, string>) => void };

function formReducer(state: any, action: FormAction) {
  switch (action.type) {
    case 'SET_FIELD':
      return { ...state, [action.field]: action.value };
    case 'SET_FIELDS':
      return { ...state, ...action.fields };
    default:
      return state;
  }
}

function useForm(initialValues = {}) {
  const [state, dispatch] = useReducer(formReducer, initialValues);
  
  const getFieldValue = useCallback((field: string) => state[field], [state]);
  
  const setFieldsValue = useCallback((fields: Record<string, any>) => {
    dispatch({ type: 'SET_FIELDS', fields });
  }, []);
  
  const setFieldValue = useCallback((field: string, value: any) => {
    dispatch({ type: 'SET_FIELD', field, value });
  }, []);
  
  return {
    state,
    getFieldValue,
    setFieldValue,
    setFieldsValue,
    // ...validate, reset, submit 等
  };
}
```

## 📊 ahooks 核心设计哲学

| 设计原则 | 说明 | 示例 |
|----------|------|------|
| **副作用封装** | 将 DOM/BOM 操作封装成声明式 Hook | `useEventListener`, `useClickAway` |
| **状态不可变** | 所有状态更新返回新引用 | `useSet`, `useMap` |
| **引用稳定** | 通过 `useRef` + `useMemo` 保证函数/对象引用稳定 | `useMemoizedFn` |
| **自动清理** | 所有副作用自动处理 cleanup | `useEventListener` 自动 remove |
| **类型安全** | 完善的 TypeScript 类型推导 | 泛型 + 重载签名 |
| **性能优先** | 避免不必要的渲染 | `useDeepCompareEffect` 深度比较 |

## 💡 实战技巧

### 技巧 1：组合使用
```tsx
// 防抖搜索 + 请求取消
const [keyword, { set }] = useBoolean('');
const { data, run, cancel } = useRequest(searchAPI, { manual: true });

const debouncedSearch = useDebounceFn((value) => {
  cancel(); // 取消上一次请求
  run(value);
}, { wait: 500 });

<input 
  value={keyword}
  onChange={(e) => {
    set(e.target.value);
    debouncedSearch(e.target.value);
  }}
/>
```

### 技巧 2：自定义 Hook 封装
```tsx
// 封装一个带防抖的表单校验
function useDebouncedValidate(
  value: string,
  validator: (val: string) => Promise<boolean>,
  delay = 300
) {
  const [validating, setValidating] = useState(false);
  const [error, setError] = useState<string | null>(null);
  
  const debouncedValidate = useDebounceFn(async (val: string) => {
    setValidating(true);
    try {
      const valid = await validator(val);
      setError(valid ? null : '校验失败');
    } finally {
      setValidating(false);
    }
  }, { wait: delay });
  
  useEffect(() => {
    if (value) debouncedValidate(value);
  }, [value]);
  
  return { validating, error };
}
```
